## 2.1.1 Tipos de Malware
As violações digitais usam muitos tipos diferentes de software mal-intencionado, ou malware, para realizar suas atividades. Abreviação de “Software Malicioso” (software mal-intencionado), o malware é qualquer código que pode ser usado para roubar dados, ignorar controles de acesso, causar danos ou comprometer um sistema. Saber quais são os diferentes tipos e como eles se espalham é fundamental para contê-los e removê-los.

### Selecione os títulos para descobrir mais sobre alguns dos malwares mais comuns:

- Spyware: Projetado para rastrear e espionar você, o spyware monitora sua atividade on-line e pode registrar todas as teclas que você pressionou no teclado, bem como capturar quase todos os dados, incluindo informações pessoais confidenciais, como detalhes bancários online. O spyware faz isso modificando as configurações de segurança dos dispositivos.
Muitas vezes, o spyware se junta ao software legítimo ou aos cavalos de Tróia.

- Adware: O adware é frequentemente instalado com algumas versões de software e é projetado para fornecer anúncios automaticamente a um usuário, na maioria das vezes em um navegador da Web. Você sabe quando vê! É difícil ignorar quando você se depara com anúncios pop-up constantes na tela.
É comum o adware vir com spyware.

- Backdoor (porta dos fundos): Esse tipo de malware é usado para obter acesso não autorizado, ignorando os procedimentos normais de autenticação para acessar um sistema. Como resultado, os hackers podem obter acesso remoto a recursos em uma aplicação e emitir comandos do sistema remoto.
Um backdoor funciona em segundo plano e é difícil de detectar.

- Ransomware: Malware projetado para manter um sistema de computador, ou os dados incluídos nele, presos até que o pagamento seja feito. O ransomware geralmente funciona criptografando os dados para que você não possa acessá-los.
Algumas versões de ransomware podem tirar proveito de vulnerabilidades específicas do sistema para bloqueá-lo. O ransomware é frequentemente disseminado por e-mails de phishing que o incentivam a baixar um anexo mal-intencionado ou uma vulnerabilidade de software.

- Scareware (software do terror): Este é um tipo de malware que usa estratégia de "medo" para induzi-lo a tomar uma ação específica. O Scareware consiste principalmente em janelas com estilo de sistema operacional que aparecem para alertar que seu sistema está em risco e precisa executar um programa específico para que ele retorne à operação normal.
Se você concordar em executar o programa específico, seu sistema será infectado por malware.

- Rootkit: Esse malware foi projetado para modificar o sistema operacional para criar um backdoor, que os invasores podem usar para acessar seu computador remotamente. A 
maioria dos rootkits utiliza as vulnerabilidades do software para escalar privilégios e modificar arquivos do sistema.
Também é comum que rootkits modifiquem a computação forense do sistema e como ferramentas de monitoramento, o que os torna muito difíceis de serem detectados. Na maioria 
dos casos, um computador infectado por um rootkit precisa ser desligado e qualquer software necessário reinstalado.

- Virus: Um vírus é um tipo de programa de computador que, quando executado, se replica e se anexa a outros arquivos pendentes, como um documento, ao inserir seu próprio 
  código. A maioria dos vírus necessita de ativação do usuário final e pode ser ativada em uma hora ou em dados específicos.
  Os vírus são relativamente inofensivos, como aqueles que exibem uma imagem divertida. Ou podem ser destrutivos, como aqueles que modificam ou excluem dados.
  O vírus também pode ser programado para ser modificado e evitar a detecção. Agora, a maioria dos vírus se espalha por unidades USB, discos ópticos, compartilhamentos de 
  rede ou e-mail.

- Cavalo de troia: Esse malware realiza operações mal-intencionadas mascarando sua verdadeira intenção. Pode parecer legítimo, mas é, de fato, muito perigoso. Os cavalos de troia exploram as particularidades do usuário e são encontrados com mais frequência em arquivos de imagem, arquivos de áudio ou jogos.
Ao contrário do vírus, os cavalos de troia não se replicam automaticamente, mas agem como um engodo para enganar o software mal-intencionado e passar por usuários desavisados.

- Worms: Esse é um tipo de malware que se replica para se espalhar de um computador para outro. Ao contrário de um vírus, que requer um programa host para ser executado, os worms podem ser executados por si próprios. Além da infecção inicial do host, eles não desativam a participação do usuário e podem se espalhar rapidamente pela rede.
Os worms compartilham padrões semelhantes: eles exploram vulnerabilidades do sistema, têm uma maneira de se propagar e contêm código mal-intencionado (carga útil) para causar danos a redes ou sistemas de computador.
Os worms são responsáveis ​​por alguns dos ataques mais devastadores na Internet. Em 19 horas, o worm infectou mais de 300.000 servidores em 2001.

- Um ataque de MitM acontece quando uma fraude digital assume o controle de um dispositivo sem o conhecimento do usuário. Com esse nível de acesso, o invasor pode interceptar e capturar informações do usuário antes de transmiti-las ao destino desejado. Esses tipos de ataques costumam ser usados ​​para roubar informações financeiras.

Existem muitos tipos de malware que possuem recursos de ataque do MitM.

- Uma variação do man-in-middle, MitMo é um tipo de ataque usado para assumir o controle do dispositivo móvel de um usuário. Quando infectado, o dispositivo móvel é instruído a capturar informações do usuário e enviá-las aos invasores. O ZeuS é um exemplo de pacote de malware com recursos MitMo. Ele permite que os invasores capturem silenciosamente as mensagens SMS de verificação de duas etapas enviadas aos usuários.

## 2.2.8 Ataques de senha:
Inserir um nome de usuário e senha é uma das formas mais populares de autenticação em um site. Portanto, descobrir a senha é uma maneira fácil para os criminosos digitais obterem acesso às informações mais importantes.

Selecione os passos para descobrir mais sobre alguns dos ataques de segurança de senha comuns.
- **Pulverização de senhas:** Essa técnica tenta obter acesso a um sistema ao “pulverizar” algumas senhas usadas com frequência em um grande número de contas. Por exemplo, um criminoso digital usa 'Password123' com muitos nomes de usuário antes de tentar novamente com uma segunda senha comumente usada, como 'qwerty'.
Essa técnica permite que o crime não seja detectado ao evitar bloqueios de conta frequentes.
## **Ataque de Pulverização de Senhas (Password Spraying): Explicação Detalhada**  

A **pulverização de senhas** (Password Spraying) é uma técnica utilizada por hackers para obter acesso a sistemas, explorando senhas fracas e comumente utilizadas. Diferente de ataques de força bruta tradicionais, que testam diversas combinações de senha em uma única conta até encontrar a correta, a pulverização de senhas faz o oposto:  

- O atacante tenta uma **mesma senha popular** em **várias contas diferentes**.  
- Depois de um tempo, tenta outra senha comum em todas as contas novamente.  
- Isso **evita múltiplas tentativas em uma mesma conta**, reduzindo o risco de bloqueios automáticos.  

### **Por que essa técnica funciona?**  
A maioria dos sistemas de autenticação tem proteções contra ataques de força bruta, como bloqueios temporários após várias tentativas de login incorretas. No entanto, **esses bloqueios geralmente são ativados por tentativas falhas consecutivas em uma mesma conta**. Ao distribuir as tentativas entre várias contas, o hacker consegue testar senhas sem levantar suspeitas.  

---

## **Passo a Passo do Ataque**  

1. **Coleta de credenciais**  
   - O atacante reúne uma lista de nomes de usuário ou endereços de e-mail.  
   - Esses dados podem ser obtidos em vazamentos anteriores, redes sociais ou ataques de phishing.  

2. **Seleção de senhas comuns**  
   - O hacker escolhe um conjunto de senhas populares, como:  
     - `123456`  
     - `Password123`  
     - `qwerty`  
     - `welcome1`  
   - Essas senhas são estatisticamente mais prováveis de serem usadas por usuários que não seguem boas práticas de segurança.  

3. **Tentativas distribuídas**  
   - O atacante tenta a **primeira senha** em todas as contas da lista.  
   - Espera um período de tempo para evitar sistemas de detecção.  
   - Depois, tenta a **segunda senha** em todas as contas, e assim por diante.  

4. **Acesso bem-sucedido**  
   - Se um login for bem-sucedido, o hacker pode acessar a conta e explorar informações sensíveis, modificar dados ou usá-la como ponto de entrada para ataques mais sofisticados.

## **Três Exemplos Práticos de Ataques de Pulverização de Senhas**  

### **1. Ataque contra contas do Office 365 e Outlook Web Access (OWA)**  
**O que aconteceu?**  
- Hackers usaram listas de e-mails de grandes empresas e tentaram logins com senhas simples como `Summer2023!` e `Welcome123`.  
- Como as organizações geralmente exigem senhas complexas, mas não as renovam regularmente, muitas contas ainda usavam senhas previsíveis.  
- Algumas contas foram comprometidas sem que os donos percebessem, e os hackers conseguiram roubar e-mails confidenciais.  

**Lição para a cibersegurança:**  
✅ Implementar **autenticação multifator (MFA)** para evitar acessos não autorizados.  
✅ Forçar o uso de **senhas longas e exclusivas**, proibindo senhas comuns.  

---

### **2. Ataque contra contas do Spotify**  
**O que aconteceu?**  
- Em um ataque massivo, criminosos usaram listas de e-mails vazados de outras plataformas e aplicaram **password spraying** com senhas fracas.  
- Como muitas pessoas reutilizam senhas, os hackers conseguiram acesso a milhares de contas.  
- As contas foram usadas para vender acessos ilegais e aplicar golpes.  

**Lição para a cibersegurança:**  
✅ **Não reutilizar senhas** entre diferentes serviços.  
✅ **Usar um gerenciador de senhas** para criar senhas fortes e únicas.  

---

### **3. Ataque contra infraestrutura governamental dos EUA**  
**O que aconteceu?**  
- Agências governamentais foram alvo de um ataque em que hackers tentaram senhas padrão como `P@ssw0rd` e `Welcome2024`.  
- Como os atacantes testaram uma senha por vez em diversas contas, o ataque passou despercebido por semanas.  
- Algumas contas foram comprometidas, permitindo acesso a sistemas internos.  

**Lição para a cibersegurança:**  
✅ Monitorar **tentativas de login suspeitas**, mesmo quando falham.  
✅ Usar **políticas de bloqueio adaptáveis**, que identifiquem tentativas de login vindas de locais e horários incomuns.

## **Conclusão**  

O ataque de **pulverização de senhas** é uma técnica furtiva e eficaz contra sistemas que não aplicam medidas de segurança robustas. Para mitigar esse ataque, recomenda-se:  
✅ Implementar **autenticação multifator (MFA)** para impedir acessos não autorizados.  
✅ Aplicar **políticas de senha rigorosas**, proibindo o uso de senhas fracas e reutilizadas.  
✅ **Monitorar tentativas de login incomuns**, mesmo se espaçadas no tempo.  
✅ Treinar usuários sobre **boas práticas de segurança**, incentivando o uso de senhas exclusivas e aleatórias.  

Com essas práticas, as organizações podem reduzir significativamente o risco de ataques desse tipo. 🚀

- **ataque de dicionário:** Um hacker tenta sistematicamente cada palavra de um dicionário ou uma lista de palavras usadas com frequência como senha, na tentativa de invadir uma conta protegida por senha.

- **Ataques de força bruta:** A forma mais simples e mais usada de obter acesso a um site protegido por senha, os ataques de força bruta veem um invasor usando todas as combinações possíveis de letras, números e símbolos no espaço de senha até acertar.

- **Ataques do arco-íris:** As senhas em um sistema de computador não são armazenadas como texto sem formatação, mas como valores de hash (valores numéricos que identificam exclusivamente os dados). Uma tabela do arco-íris é um grande dicionário de hashes pré-computados e as senhas das quais eles foram calculados.
Ao contrário de um ataque de força brutal que precisa calcular cada hash, um ataque do arco-íris compara o hash de uma senha com os armazenados na tabela do arco-íris. Quando um invasor encontra uma correspondência, ele identifica a senha usada para criar o hash.

## **Ataque com Tabelas do Arco-Íris: Explicação Detalhada**  

Os sistemas de autenticação não armazenam senhas em texto puro por razões de segurança. Em vez disso, eles usam **funções de hash criptográficas** para transformar as senhas em valores únicos e irreversíveis. Quando um usuário digita a senha, o sistema aplica a mesma função de hash e compara o resultado com o hash armazenado.  

O **ataque com tabelas do arco-íris** (Rainbow Table Attack) explora essa técnica de armazenamento ao usar um banco de dados massivo contendo pares **(senha -> hash)** pré-computados. Isso permite que o invasor **compare rapidamente** um hash roubado com os registros dessa tabela para descobrir qual senha o gerou.  

### **Funcionamento do Ataque**
1. **Criação da Tabela do Arco-Íris**  
   - O atacante gera uma grande tabela contendo várias senhas comuns e seus respectivos hashes.  
   - As senhas são processadas por uma **função de hash** e armazenadas junto com seus valores resultantes.  

2. **Roubo de Hashes**  
   - O invasor obtém hashes de senhas de um banco de dados comprometido.  
   - Esses hashes podem ser extraídos por meio de ataques como **SQL Injection** ou exploração de falhas de segurança.  

3. **Comparação de Hashes**  
   - O atacante compara os hashes roubados com os presentes na tabela do arco-íris.  
   - Se encontrar uma correspondência, ele descobre a senha correspondente sem precisar calcular novos hashes.  

### **Vantagem Sobre Força Bruta**  
Diferente de um ataque de força bruta, que testa todas as combinações de senha uma a uma e recalcula hashes em tempo real, a tabela do arco-íris já possui hashes prontos. Isso acelera significativamente o ataque, tornando-o mais eficiente.  

---

## **Três Exemplos Práticos de Ataques com Tabelas do Arco-Íris**

### **1. Vazamento de Hashes do LinkedIn (2012)**  
**O que aconteceu?**  
- Hackers invadiram servidores do LinkedIn e roubaram aproximadamente **117 milhões de credenciais de usuários**.  
- Os hashes das senhas estavam protegidos com SHA-1, mas sem **salting** (um método de adicionar um valor aleatório antes de aplicar o hash).  
- Como o SHA-1 é uma função vulnerável, os atacantes usaram tabelas do arco-íris para recuperar muitas senhas.  

**Lição para a cibersegurança:**  
- Utilizar hashes seguros, como **bcrypt, Argon2 ou PBKDF2**, que incluem um fator de complexidade para dificultar ataques.  
- Sempre usar **salts aleatórios**, pois isso impede o uso de tabelas pré-computadas.  

---

### **2. Vazamento da PlayStation Network (2011)**  
**O que aconteceu?**  
- A PlayStation Network da Sony foi comprometida, expondo informações de **77 milhões de usuários**, incluindo credenciais de login.  
- O ataque revelou que muitas senhas estavam armazenadas sem hash ou proteção adequada.  
- Como muitas pessoas reutilizam senhas em vários serviços, os hackers puderam usar as credenciais para invadir outras contas.  

**Lição para a cibersegurança:**  
- Empresas devem garantir que senhas sejam armazenadas com **métodos de hashing seguros**.  
- Usuários devem usar **gerenciadores de senha** para evitar reutilização em diferentes plataformas.  

---

### **3. Ataque ao Yahoo (2016)**  
**O que aconteceu?**  
- Um dos maiores vazamentos da história: **3 bilhões de contas comprometidas**.  
- Muitas senhas estavam protegidas apenas com o algoritmo **MD5**, que é extremamente fraco e facilmente quebrável com tabelas do arco-íris.  

**Lição para a cibersegurança:**  
- Evitar **MD5 e SHA-1**, pois são inseguros contra ataques modernos.  
- Implementar métodos como **hashing com salting e múltiplas iterações** para aumentar a resistência contra ataques de pré-computação.  

---

## **Conclusão**  
O ataque de tabela do arco-íris é uma ameaça real e eficaz contra senhas armazenadas sem boas práticas de segurança. Para mitigar esse tipo de ataque, recomenda-se:  
✅ Usar funções de hash seguras (**bcrypt, Argon2, PBKDF2**).  
✅ Aplicar **salts aleatórios** para cada senha.  
✅ Exigir senhas longas e complexas para dificultar ataques.  
✅ Monitorar e proteger bancos de dados para evitar vazamentos.  

Isso reduz drasticamente a eficácia desse ataque, tornando a quebra de senhas muito mais difícil para os invasores. 🚀

- **Interceptação de tráfego** Texto sem formatação ou senhas sem criptografia podem ser facilmente lidas por outras pessoas e máquinas interceptando comunicações.
Se você armazenar uma senha em texto claro e legível, qualquer pessoa que tenha acesso à sua conta ou dispositivo, autorizado ou não autorizado, poderá lê-la.
